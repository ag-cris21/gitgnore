## 📋 Descripción

Este proyecto tiene como objetivo aprender a utilizar el archivo `.gitignore` en un repositorio de Git. Se muestra cómo ignorar archivos y carpetas que no deben ser rastreados por el sistema de control de versiones, manteniendo el repositorio limpio y organizado.

## 🎯 Objetivos

- Comprender qué es y para qué sirve el archivo `.gitignore`
- Aplicar patrones de exclusión correctamente
- Mantener un historial de commits limpio y profesional
- Practicar el flujo de trabajo colaborativo con Git y GitHub

## 🗂️ Estructura del Proyecto

```
gitgnore/
├── .gitignore          # Archivo de exclusiones de Git
├── README.md           # Documentación del proyecto
├── index.html          # Página principal
├── components/
│   ├── navbar.html     # Componente de navegación
│   └── footer.html     # Componente de pie de página
├── css/
│   └── styles.css      # Estilos del proyecto
├── js/
│   └── script.js       # Lógica JavaScript
└── pages/
    └── welcome.html    # Página de bienvenida
```

## 📌 Convención de Commits

Este proyecto sigue la convención de [Conventional Commits](https://www.conventionalcommits.org/):

| Tipo | Uso |
|------|-----|
| `feat` | Nueva funcionalidad |
| `fix` | Corrección de errores |
| `docs` | Cambios en documentación |
| `chore` | Tareas de mantenimiento |
| `style` | Cambios de estilo/formato |
